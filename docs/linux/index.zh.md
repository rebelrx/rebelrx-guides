## 🚫 为什么避免使用 systemd

现代 Linux 发行版在很大程度上已经将 **systemd** 标准化为其初始化系统。

RebelRx 明确避免使用 systemd。

### 为什么？

因为 systemd 代表了一种转变：

* 核心系统控制的集中化
* 系统组件之间的深度集成
* 相较于传统 UNIX 风格初始化系统，透明度降低

更重要的是：

> 它引入了一个可以大规模嵌入外部控制机制的层。

这包括当前全球范围内不断推进的：

* 年龄验证系统
* 与身份绑定的访问控制
* 设备级强制执行机制

RebelRx 的假设正相反：

> 操作系统正在成为执行控制的层。

而 systemd 是 Linux 中最可能的插入点。

---

## 🔐 隐私与控制理念

在所有主流平台（Windows、macOS、iOS、Android）中，我们已经看到：

* 强制账户集成
* 日益增加的遥测与跟踪
* 与身份绑定的平台级限制

Linux 通常被视为替代方案。

但是：

> 并非所有 Linux 都是一样的。

你的控制程度在很大程度上取决于：

* 你选择的发行版
* 你的初始化系统
* 你是否愿意避免以便利为导向的默认设置

---

## 🧱 推荐方案（非 systemd）

为了尽量减少暴露于集中控制层，本指南重点推荐：

### 🔹 Artix Linux（基于 Arch）

* 无 systemd
* 滚动更新
* 多种初始化选项（OpenRC、runit、s6）
* 最大程度的灵活性与控制

### 🔹 Devuan（基于 Debian）

* 无 systemd
* 稳定发布模型
* 熟悉的 Debian 生态
* 适用于服务器和长期部署

👉 非 systemd Linux 发行版完整列表：<https://nosystemd.org/>
